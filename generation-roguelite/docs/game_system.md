## A. 世代交代サイクルの完成

### A-1. 死亡と弔い演出
1. キャラクターが寿命に達した時、または致命的な障害物衝突で死亡した時に、ゲーム画面をゆっくりフェードアウトさせる
2. フェードアウト後に「弔い画面」を表示する。この画面には以下を含む:
   - 故人の名前、到達年齢、人生パス
   - 3枚のハイライトスナップショット（そのキャラの人生で起きたイベントから重要度が高い順に3つ選び、イベントテキストを短縮して表示。画像は仮のアイコンやシンボルで可）
   - 死因の表示（「老衰」「障害物衝突」「イベント失敗による致命傷」等）
   - 墓石のグラフィック（仮のシンプルな矩形+十字で可）
   - 1世代のスコア表示（到達年齢 × 能力値合計 × イベントクリア数）
3. 弔い画面に2〜3秒滞在した後、「次の世代へ」ボタンを表示
4. この弔い画面のタイミングでインタースティシャル広告の挿入ポイントを設ける（実際の広告SDKは未統合で可、AdManager.csにプレースホルダーメソッドを用意）
   - 広告表示判定: 前回広告表示からの経過世代数 >= 2 かつ 経過プレイ時間 >= 10分

### A-2. 次世代キャラクターの生成
1. 「次の世代へ」ボタン押下後、次世代キャラクター生成画面を表示
2. 生成画面では以下を表示:
   - 親から引き継ぐ遺産額（親の財力に基づく）
   - 子の種類: 実子（親の能力値を50%遺伝）または養子（遺伝なし、ランダムでユニークスキル1つ付与）
   - 子の初期能力値プレビュー
3. プレイヤーが実子/養子を選択して「誕生」ボタンを押すと新しい世代が始まる
4. 遺産に応じた幼少期加速: 遺産額が一定値以上なら幼少期の時間経過速度が2倍〜3倍になる
5. リワード広告の挿入ポイント: 「誕生」ボタンの横に「広告を見て遺産ボーナス」ボタンを配置（プレースホルダー。押すと遺産に固定値+10〜20を加算）

### A-3. 世代データの蓄積
1. GenerationHistory クラスを作成し、各世代の以下データを保存:
   - 名前、到達年齢、死因、人生パス、最終能力値、1世代スコア、クリアしたイベント一覧、装備リスト
2. 累計スコアの計算: 全世代のスコアを合算し、連続世代数に応じたボーナス倍率を掛ける
   - ボーナス倍率: 1世代目=1.0x、2世代目=1.1x、3世代目=1.2x ... 10世代以降は2.0x上限
3. 家系が途絶えた場合（伴侶なし・子なしで死亡）はボーナス倍率がリセットされる

### A-4. 伴侶システム
1. 壮年期（31〜55歳）にランダムで「伴侶候補イベント」が発生する
2. 伴侶候補は名前、能力値（3つの能力がランダム）、特性（1つ）を持つ
3. 魅力の判定で求婚の成否が決まる（魅力が高いほど成功率UP）
4. 伴侶がいない状態で老年期に入ると「養子を迎える」選択が自動的に強制される
5. 伴侶がいない状態で死亡すると家系断絶 → ボーナスリセット → 新しい家系を始祖から開始

## B. 人生フェーズごとのアクション分化

### B-1. フェーズ別操作の実装
現在のタップ/スワイプ処理を拡張し、人生フェーズごとに動作を切り替える。

幼少期（0-12歳）:
- タップ → 「好奇心タッチ」: 画面上のキラキラ光るオブジェクト（CollectibleNode）をタップで拾う。拾うと小さな能力値ボーナスまたは伏線アイテムを獲得
- スワイプ → 無効（何も起きない）
- 障害物は出現しない
- CollectibleNodeは3〜5秒間隔でランダムな画面位置に出現し、2秒で消える

青年期（13-30歳）:
- タップ → 「ジャンプ」（現在の実装を維持）
- スワイプ上 → 無効
- スワイプ左右 → 「攻撃」: 前方の敵タイプ障害物を破壊。破壊すると小さなドロップ（ゴールドまたはアイテム）
- 障害物の移動速度は基本速度の1.0倍

壮年期（31-55歳）:
- タップ → 「防御（受け流し）」: タイミングよくタップすると障害物のダメージを80%カット。判定窓は障害物接触の0.3秒前〜接触の瞬間
- スワイプ左右 → 「攻撃」（青年期と同じ）
- ジャンプ不可
- 障害物の移動速度は基本速度の1.2倍
- 障害物による寿命減少量は青年期の2倍

老年期（56歳〜寿命）:
- タップ → 「杖をつく」: 一時的に歩行速度が0.5倍に減速し、3秒間障害物の出現を抑制
- スワイプ左右 → 「回避」: キャラクターが上下に小さく移動して障害物を避ける
- 攻撃手段なし
- 障害物の移動速度は基本速度の0.8倍だが、1回の衝突で寿命-5年以上（致命的）
- 障害物の出現頻度自体は低い

### B-2. フェーズ切り替え演出
1. フェーズが切り替わる瞬間に0.5秒のスローモーション演出
2. 画面端からフェーズ名がスライドイン表示（「— 青年期 —」等）
3. ナビキャラの吹き出しにフェーズ変化コメントを表示（テンプレートから選択）
   - 幼少期→青年期: 「大人の世界へようこそ！」「さあ、冒険の始まりだよ！」等
   - 青年期→壮年期: 「あれ、ジャンプが…もう無理みたい」「経験が力になる時期だね」等
   - 壮年期→老年期: 「無理しないでね…」「静かに、でも確実に歩もう」等
4. キャラクターのスプライト/色が変化してフェーズを視覚的に区別（仮実装: ColorRectの色を変える）

## C. ハクスラ・装備システム

### C-1. 装備データ構造
1. Equipment クラスを作成:
   - name: string（装備名）
   - rarity: enum（Common, Uncommon, Rare, Epic, Legendary）
   - stat_bonuses: Dictionary<StatType, int>（能力値ボーナス）
   - lifespan_modifier: int（寿命への影響、正負あり）
   - is_heirloom: bool（家宝として世代を超えて引き継ぐか）
   - era: string（ドロップ可能な時代）
2. レアリティごとのドロップ確率: Common=60%, Uncommon=25%, Rare=10%, Epic=4%, Legendary=1%
3. 「運」の能力値に応じてレアリティテーブルを補正（運が高いほどレア以上の確率UP）

### C-2. 装備のドロップと管理
1. イベント成功時にドロップ判定を行う（成功イベントの50%でドロップ発生）
2. ドロップ時にレアリティ演出: Common=白テキスト、Uncommon=緑、Rare=青、Epic=紫、Legendary=金+画面フラッシュ
3. インベントリは最大10枠。装備中は別枠で3つまで（武器/防具/アクセサリー）
4. インベントリが満杯の場合、新しいドロップと既存装備の入れ替え画面を表示
5. 装備の能力値ボーナスはキャラクターのステータスにリアルタイム反映

### C-3. 家宝システム
1. 遺言画面で装備1つを「家宝」に指定可能
2. 家宝はレア以上のレアリティのみ指定可能
3. 家宝は次世代に引き継がれ、幼少期から装備可能
4. 家宝には世代を重ねるごとに小さなボーナスが蓄積される（+1/世代）
5. 家宝一覧はコレクション画面で確認可能

実装時の注意事項
全てのUIはモバイル画面（縦持ち想定ではなく横持ち、16:9）に収まるように設計すること
タッチ操作が主であることを前提にボタンサイズは最低48x48dp以上
仮グラフィックは全てコードで生成可能なもの（ColorRect、Label、簡易図形描画）で可。外部画像アセットは不要
パフォーマンスを意識し、不要なノードは queue_free() で解放すること
各機能はできるだけ独立したクラス/シーンに分離し、docs/game_design.md のプロジェクト構造に従うこと
