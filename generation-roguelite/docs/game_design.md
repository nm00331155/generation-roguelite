# 世代継承ローグライト — プロジェクト仕様書 v2.1

## 1. プロジェクト概要

### 1.1 コンセプト
2D横スクロールで主人公が自動的に右へ歩み続ける「人生」シミュレーションゲーム。
時間経過で年齢・能力が変化し、ローカルSLM（小規模言語モデル）が動的にイベントを生成する。
老衰までに次世代へバトンを繋ぎ、世代を重ねて累計スコアを最大化する。
ローグライト × ハクスラ × セミオートRPGのハイブリッド。

**コアコンセプト**: 「人生は待ってくれない」— 時間は止まらず、選択は常に迫られる。

### 1.2 ターゲット
- カジュアルゲーマー〜中級ゲーマー
- 周回・育成・コレクション好きの層

### 1.3 ジャンル
世代継承型ローグライト / ハクスラ / セミオートRPG

### 1.4 プラットフォーム
Android

## 2. 技術構成

### 2.1 開発環境
- **ゲームエンジン**: Godot Engine .NET版（最新安定版）
- **スクリプト言語**: C#
- **エディタ**: VSCode + C# Dev Kit + godot-tools
- **AI支援**: GitHub Copilot Premium
- **バージョン管理**: GitHub（Privateリポジトリ）

### 2.2 ローカルSLM
- **モデル**: Qwen3 1.7B（4bit量子化、約1GB）
- **推論エンジン**: llama.cpp（NDKビルド）
- **C#からの呼び出し**: P/Invoke（ネイティブ関数呼び出し）
- **イベント管理**: 事前キャッシュ（20〜30個先行生成）+ リアルタイム生成の併用
- **フォールバック**: JSONパース失敗時はテンプレートイベントを使用

### 2.3 音声
- **合成エンジン**: AivisSpeech
- **音声モデルライセンス**: ACML 1.0 または CC0 のモデルのみ使用（ACML-NCは商用不可のため使用禁止）
- **ファイル形式**: OGG Vorbis（WAVから変換、容量1/10）
- **管理方式**: セリフIDに対応した命名規則（例: navi_event_success_01.ogg）
- **追加ナビキャラ音声**: 購入時に追加ダウンロード（初期アプリサイズに含めない）

### 2.4 アプリサイズ目安
約1.5〜2GB（ゲームアセット + SLMモデル込み）

## 4. 人生フェーズシステム

### 4.1 幼少期（0〜12歳）
- 遺産が豊富なら加速スキップ可能
- あえてプレイすれば「師匠イベント」でユニークスキルの種を獲得可能
- 障害物はほぼなし（守られている演出）
- **操作**: タップのみ →「好奇心タッチ」（道端のアイテム拾い、生き物に触れる等）
- 拾ったものが後の人生に影響する伏線になる
- チュートリアル期間を兼ねる

### 4.2 青年期（13〜30歳）
- 能力値の伸びが最大のゴールデンタイム
- イベント密度最大
- 「就職」「冒険」「修行」等の**人生パス分岐**を1回選択 → 以降のイベントプールが変化
- **操作**: タップ=「ジャンプ」、スワイプ=「攻撃」
- 障害物・敵が多く、アクション密度のピーク

### 4.3 壮年期（31〜55歳）
- 能力値は横ばいだがイベント報酬のレア度が上昇
- 「病気」「事故」等のリスクイベント増加
- 伴侶探しのピーク期間（装備掘りと次世代準備のジレンマ）
- **操作**: タップ=「防御（受け流し）」、スワイプ=「攻撃」
- ジャンプ不可（「もう若くないから飛べないが、経験で受け止める」）
- 障害物の寿命減少が大きくなる

### 4.4 老年期（56歳〜寿命）
- 能力値が徐々に低下
- 「遺言」UIが解放 → 長く生きるほど設定可能項目が増える
  - どの装備を家宝にするか
  - 遺産の配分比率
  - 子への最後の教え（バフ1つ選択）
- **操作**: タップ=「杖をつく（加速/安定化）」、スワイプ=「回避」
- 攻撃手段なし。障害物は少ないが1回で致命的

## 5. 操作方式

### 5.1 基本操作
- **タップ**と**スワイプ**の2種類のみ
- 年代によって同じ操作の意味が変化（操作の学習コストゼロで体験が変わる）

### 5.2 自動/手動の切り分け
- 通常時: セミオート歩行 + 障害物回避 + イベント選択
- 画面を開いていればセミオートで進行（眺めているだけでも一応進む）
- プレイヤーが介入するほど良い結果を得られる

### 5.3 オフライン時
- **アプリを閉じると時間凍結**（年齢・歩行は一切進行しない）
- オフライン時間に応じて「家系の評判ポイント」がゆっくり蓄積
  - ショップ品揃え改善、伴侶候補の質に影響
  - 年齢・イベント進行には一切関与しない

## 6. イベントシステム

### 6.1 SLM生成

ローカルSLMがJSON形式でイベントを動的生成する。

**入力パラメータ**:
- age（年齢）
- stats（能力値: 体力/知力/魅力/運/財力）
- inventory（所持品）
- generation（世代数）
- era（時代）
- life_path（青年期に選んだ人生パス）
- bonds（縁のあるNPC一覧、最大3名）
- family_traits（家系に蓄積された特徴）
- recent_events（直近3イベントの要約）
- era_mechanic（時代固有メカニクス名）

**出力JSON例**:
```json
{
  "event_text": "山賊の縄張りに足を踏み入れた。",
  "choices": [
    {
      "text": "戦う",
      "check": "体力",
      "difficulty": 12,
      "success": "山賊を撃退した。財力+8",
      "fail": "深手を負った。体力-5, 寿命-2年"
    },
    {
      "text": "交渉する",
      "check": "魅力",
      "difficulty": 15,
      "success": "見逃してもらえた。運+2",
      "fail": "交渉決裂。財力-10"
    },
    {
      "text": "逃げる",
      "check": null,
      "success": "何とか逃げ切った。",
      "fail": null
    }
  ],
  "terrain": {
    "obstacle_density": 0.7,
    "obstacle_type": "enemy",
    "speed_modifier": 1.2
  }
}
6.2 制限時間
時間は止まらない（コアコンセプト「人生は待ってくれない」）
制限時間はテキスト長に応じて動的調整（7〜12秒目安）
時間切れの場合は最もリスクの低い選択肢が自動選択
6.3 品質管理
生成イベントに対する機械的スコアリング:

選択肢数が2以上あるか
成功/失敗で異なる結果が設定されているか
能力値変動が±10以内に収まっているか
テキストが20文字以上120文字以下か
閾値以下は破棄して再生成
パース失敗時はテンプレートイベントを使用
空き時間に20〜30個を事前キャッシュ
6.4 地形パラメータ連動
SLMがイベント生成時に terrain パラメータも出力
これを障害物生成アルゴリズムに渡し、物語とアクションを連動させる
山賊イベントなら障害物が多く、穏やかな村なら少ない
7. 障害物・難易度システム
7.1 基本ルール
障害物回避失敗で寿命が減少する
年齢フェーズによって影響度が変化:
幼少期: ほぼ無害
青年期: 小ダメージ
壮年期: 中ダメージ
老年期: 致命的
7.2 二層構造の難易度制御
レイヤー1: SLMによる物語的難易度

イベント生成時の terrain.obstacle_density が基準値
レイヤー2: DDA（Dynamic Difficulty Adjustment）アルゴリズム

直近30秒間の障害物回避成功率を常時トラッキング
成功率90%以上 → 障害物密度を徐々に上げる
成功率50%以下 → 密度を下げる
SLMの基準値に対し±0.2の範囲で微調整
年齢係数:

同じ obstacle_density でも年齢によって体感難易度が変化
青年期: 障害物の移動速度が遅め（避けやすい）
老年期: 速め（致命的）
8. ナビゲーターキャラクター
8.1 役割
プレイヤーの人生をナビゲートする常駐キャラクター
イベント時の簡略メッセージ表示（選択肢を一言で要約）
要所でコメント・雑談・愚痴を挟む
頻繁にはしゃべらない（バランス重視）
イベントの内容や結果に反応 / たまに勝手に話す程度
8.2 発話タイミング例
イベント発生時: 選択肢の一言要約（「助ける？ 無視する？ 旅人が倒れてるよ！」）
イベント成功/失敗時: リアクション（「やったね！」「うわ…」）
年齢フェーズ変化時: 一言（「あれ、ジャンプが…もう無理みたい」）
ランダムな雑談（低頻度）
8.3 好感度曲線
最初はよそよそしい → 徐々に打ち解ける → 長くプレイすると裏設定が見える
ナビキャラ自体がリテンション装置として機能
8.4 実装方式
セリフはSLM生成ではなく、テンプレート＋条件分岐で事前作成
イベント結果タグ（成功/失敗、能力値変動方向等）に応じた複数パターン
音声: AivisSpeechで事前生成したOGGファイル（カテゴリ+連番でランダム再生）
想定セリフ規模: 200〜500セリフ程度
8.5 課金要素
ナビキャラの差し替えが可能（課金）
キャラごとに異なる人格・口調・リアクション
追加キャラの音声セットは購入時に追加ダウンロード
9. 世代交代ルール
9.1 基本ルール
子は常に0歳からスタート
親世代の遺産（財力・アイテム）に応じて幼少期が加速
遺産が豊富なほど「有効プレイ時間」が長くなる
9.2 実子 vs 養子
実子: 親の能力値を一部遺伝（遺伝ボーナスあり）
養子: 遺伝ボーナスなし、代わりにユニークスキル持ちの可能性
「いつ子を作るか」「実子 vs 養子」に戦略性
9.3 弔い演出
死亡時に短い演出（2〜3秒）
その人生のハイライトが3枚のスナップショットとして表示
例: 師匠との出会い、結婚、最大の冒険
墓石が建つ
新しい赤ん坊の泣き声とともに次世代開始
9.4 遺言システム（老年期限定）
老年期に解放
長く生きるほど設定可能項目が増加:
どの装備を家宝にするか
遺産の配分比率
子への最後の教え（バフ1つ選択）
10. 能力値
能力値	効果
体力	寿命・耐久に影響
知力	イベント判定の成功率
魅力	伴侶発見率・イベント選択肢増加
運	レアイベント・レアドロップ出現率
財力	ショップ利用・遺産額・幼少期スキップ速度
11. ハクスラ要素
11.1 装備ドロップ
イベント報酬として装備品・特性がドロップ
例: 「勤勉の指輪: 知力+3」「呪いのペンダント: 運+5, 寿命-5年」
11.2 レアリティ
コモン → アンコモン → レア → エピック → レジェンダリー

11.3 家宝システム
一部装備は世代を超えて引き継ぎ可能
レジェンダリー収集が長期目標
12. 「縁」システム
伴侶以外のNPCとの「縁」を蓄積
助けた旅人が後に再登場して助けてくれる等
SLMプロンプトに「既知NPC一覧（bonds）」を渡して実現
縁が深いNPCは「盟友」になる
戦闘加勢
レアアイテム取引の相手
世代を跨いで「親の盟友の子孫」として再登場する可能性あり
13. メタ進行: 「家訓」システム
世代を重ねるごとに「家訓」を1つずつ制定可能
累計スコアの一部を消費して制定
最大20個程度
家系が途絶えても家訓はリセットされない（永続的メタ進行）
家訓の例
「質素倹約の家訓」: 毎世代開始時に財力+5
「武芸奨励の家訓」: 幼少期に体力成長イベント確率+20%
「婚姻重視の家訓」: 魅力判定に永続+2
14. 時代変化
14.1 時代遷移
一定世代ごとに時代が遷移: 原始 → 古代 → 中世 → 近世 → 現代 → 未来

SLMプロンプトに era を渡してイベントの世界観が変化。 将来の大型アップデートで「異世界編」「SF編」等の世界観拡張も視野（初期リリースには含めない）。

14.2 時代固有メカニクス（各時代にUIパーツ1つ追加）
時代	メカニクス	内容
原始	狩猟	タイミングタップ戦闘主体
古代	政治	権力闘争の選択肢イベント増加
中世	信仰	信仰ゲージによるバフ/デバフ変動
近世	発明	素材から装備クラフト
現代	投資	財力で資産運用、リスクとリターン
未来	遺伝子改造	能力値の直接書き換え（デメリットあり）
15. 災厄と黄金期
数世代に1回、世代レベルの大規模イベント
災厄:「疫病」「大戦」「天変地異」等 → 世代全体の難易度上昇
黄金期:「文化的繁栄期」「大航海時代」等 → 報酬倍増
災厄を生き延びると称号が付与され、子孫に耐性ボーナスが遺伝
16. スコアリングとインフレ設計
16.1 基本スコア計算
1世代スコア = 到達年齢 × 能力値合計 × イベントクリア数 × レアアイテム補正
累計スコア  = 全世代合算（世代継続ボーナス倍率あり）
家系が途絶えるとボーナスリセット → 伴侶探しに緊張感
16.2 段階的インフレ
時代ごとにスケールのレンジが変化:

時代	能力値レンジ	スコア単位
原始	1〜50	千
古代	10〜100	万
中世	50〜500	十万
近世	100〜1000	百万
現代	500〜5000	億
未来	5000〜50000	兆
時代が進むごとに「桁が変わる」体験を提供。

実装済みパラメータ（Phase 8）:

時代	score_pivot	score_factor上限	最終係数クランプ
原始	36,000	+0.35	0.75〜2.60
古代	92,000	+0.42	0.85〜3.20
中世	210,000	+0.50	0.95〜4.10
近世	440,000	+0.58	1.05〜5.00
現代	920,000	+0.70	1.15〜6.30
未来	1,850,000	+0.85	1.25〜7.80

16.3 乗算構造
基礎能力値は加算で成長
家訓・家宝による%補正が乗算で乗る
世代を重ねるほど%補正が積み上がり、同じ能力値でもスコアが爆発的に伸びる
17. 周回・コレクション要素
17.1 家系図
全世代をツリー表示
各世代のハイライト記録
17.2 死因コレクション
「老衰」「戦死」「病死」「事故死」「暗殺」「呪い」等を図鑑収集
墓石デザインが死因に応じて変化
死因ごとに次世代へのバフ/デバフ
例: 戦死した親の子は体力+5, 運-3
17.3 実績・称号
「10世代継続」「レジェンダリー5個収集」「大疫病の生存者」等
17.4 始祖選択
初回は「平凡な始祖」のみ
累計実績に応じてアンロック: 「戦士の始祖」「商人の始祖」「呪われた始祖」等
始祖ごとに初期能力値・初期時代・固有イベントプールが異なる
17.5 「家系の伝説」自動生成
10世代ごとまたは家系断絶時にSLMが家系全体の物語を短い散文で生成・保存
SNSシェア対応
17.6 「後悔」イベント
老年期に過去の選択を振り返る演出
SLMにその世代の選択履歴を渡して自動生成
ゲーム的な影響はなし（純粋な物語演出）
18. マネタイズ
18.1 広告
強制広告（インタースティシャル）:

世代交代の暗転中に挿入
毎回ではなく2〜3世代に1回、またはプレイ時間10〜15分に1回のいずれか長い方
広告除去購入者には非表示
リワード広告（任意視聴）:

世代交代時の遺産ボーナス（次世代の初期財力+10、1世代1回）
イベント失敗後のやり直し（1世代1回限定）
ショップの追加商品枠開放（+1枠、当世代のみ）
18.2 課金アイテム
広告除去（980円買い切り）:

強制広告を全て除去
リワード広告は引き続き任意視聴可能で効果も維持される旨を明記
ナビゲーターキャラ変更（各480円）:

キャラごとに異なる人格・口調・リアクション
音声セット付き
「運命の書」（ミニバトルパス、360円 / 2週間）:

無料トラックと有料トラック
世代を重ねるごとにポイントが溜まり報酬解放
無料トラック: 通常装備、ゴールド、消費アイテム
有料トラック: 限定家紋、限定墓石デザイン、レアリティ確定チケット、限定称号等
コスメティック（家紋・墓石・背景テーマ等）:

基本はゲーム内実績・コレクションで解放
一部の限定コスメ（季節限定等）はバトルパス有料トラックに含める形で収益化

## 技術構成（更新: 2026-02-27）

### 確定環境
- Engine: Godot 4.6.1 .NET (Mono)
- Language: C#
- .NET SDK: 8.0.x（ビルド用）＋ 10.0.x（共存、Godotは8を使用）
- JDK: Adoptium Temurin 17（Gradle互換のため。Godot同梱のJDK 25は使用しない）
- Android SDK: commandline-tools のみ（Android Studio不使用）
- Android target: API 34, minimum API 24
- adb: C:\Users\mikami\AppData\Local\Android\Sdk\platform-tools\adb.exe
- 画面: 1080×1920 縦持ち固定 (portrait)
- Renderer: Mobile (Compatibility)

### エクスポート設定
- Export preset: "Android"
- Package: com.yourname.inheritance（仮）
- Orientation: portrait
- Gradle build: 有効（android/ フォルダ内にビルドテンプレート）
- Debug keystore: Godot 自動生成

### 既知の注意事項
- C#/.NET Android export は Godot 公式で「実験的」扱い
- ParallaxBackground は 4.6 で非推奨警告が出る（動作に問題なし）
- Godot 同梱 JDK 25 は Gradle 8.11.1 と非互換のため使用禁止
