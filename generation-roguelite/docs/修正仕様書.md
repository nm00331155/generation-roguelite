変更概要
バグ修正5件 + ゲーム性改善8件 + 見た目改善5件 + 日本語化を一括適用する。

A. 定数変更
定数	旧値	新値	理由
CellSize	280f	300f	画面拡大
CellGap	20f	12f	画面拡大
GridOriginX	100f	42f	中央寄せ
MoveDuration	0.15f	0.28f	ヌルヌル感
HitDamage	10f	30f	3回で死亡
GuardDamage	2f	15f	ガード時も痛い
新規追加 EventTimeLimit	-	3f	イベント制限時間
B. バグ修正
B1. 当たり判定をセル中心通過時に変更 UpdateObstacleMotion内の ResolveCellPass 呼び出しタイミングを変更。FloorToInt(t) ではなく t が cellIndex + 0.4 ～ cellIndex + 0.6 の範囲（セル中心付近）にいるときに判定する。具体的には各セルについて t >= cellIndex + 0.5f になった瞬間に1回だけ判定。

B2. moveアニメを方向別に分離 BuildPlayerFramesで "move" を2つに分ける:

"move_forward": フレーム0,1,2,3,4,5（右/上/下移動時）
"move_back": フレーム0,1,6,7,8（左移動時）
TryMove内で dx == -1 なら _player.Play("move_back")、それ以外は _player.Play("move_forward")。

RegisterAnimationはそのまま9フレーム登録。新たに RegisterPartialAnimation(SpriteFrames frames, string name, Texture2D sheet, float fps, bool loop, int[] frameIndices) を追加し、指定インデックスのフレームだけ登録する。

B3. 連続移動時にTweenキャンセル TryMoveの先頭で _isMoving チェックを削除。代わりに、移動中の場合は既存Tweenをキャンセルし、現在のTween完了コールバックを呼ばずに即座に位置を確定してから新しい移動を開始する。_activeTweenフィールド（Tween?型）を追加し、TryMove開始時に _activeTween?.Kill() してから新Tweenを作成。

B4. イベント中も障害物を継続 _Processで _state == GameState.WaitingChoice のときも TickObstacles(dt) を呼ぶ。_UnhandledInputで WaitingChoice状態でも移動・ガード入力を受け付ける（_state != GameState.Playing のreturnを _state is not (GameState.Playing or GameState.WaitingChoice) に変更）。

B5. ガードフレーム制御 guardアニメを loop: false に変更。_Process内に TickGuardFrameControl() を追加。_isGuarding中にフレームが5に達したら _player.Pause() でフレーム停止。EndGuard時に _player.Play() で再開（フレーム6-8が再生され、AnimationFinished後にrunへ遷移）。

C. ゲーム性改善
C1. Wave数を3→10に拡張 BuildWaveDefsを書き換え。Wave1-2は右のみ（3-4体、SpeedPerCell 0.5）、Wave3-4は右+上（5-6体、Speed 0.4）、Wave5-6は右+上+左（7-8体、Speed 0.35）、Wave7-8は全4方向（9-10体、Speed 0.3）、Wave9-10は全方向（12-15体、Speed 0.25、同時出現あり）。

C2. イベント3秒制限 _eventTimerフィールドを追加。ShowEventで _eventTimer = EventTimeLimit にセット。_Processで WaitingChoice中に _eventTimer -= dt し、0以下になったら OnEventOptionSelected(chooseA: _rng.Randf() > 0.5f) で自動選択。EventPanelにタイマー表示用の _eventTimerLabel（Label）を動的生成し、残り秒数を表示。

C3. ハイスコア保存 _highScore(int), _highWave(int) フィールド追加。_Readyで LoadHighScore() を呼び、user://highscore.json から読み込み。死亡時に現在スコア・到達Waveがハイスコアを超えていたら SaveHighScore() で書き込み。JSONは {"score":12500,"wave":7} 形式。Godot.FileAccess を使用。

C4. 死亡画面に情報表示 _deathOverlayに動的にLabel3つ追加: 「スコア: {_totalScore}」「ハイスコア: {_highScore}」「到達Wave: {_currentWaveIndex+1}/{_waves.Length}」。Die()内で設定、リスタート時にクリア。

C5. Wave間待機を短縮 TryResolveWaveEndでWaveクリア後、イベント表示までの待機を1秒に（既存のTimerCallbackかTweenのDelayを1秒に設定）。

C6. 警告番号の視認性向上 ShowWarning内の order Label: フォントサイズ36→52、色を白(Colors.White)に変更、アウトラインサイズ4・色を赤(new Color(1,0.2,0.2,1))に設定（AddThemeConstantOverride("outline_size", 4) + AddThemeColorOverride("font_outline_color", ...)）。

C7. 障害物が端で反射せず通過して消える ComputeObstaclePositionを修正。tが0〜2の範囲はセル0→セル2を線形補間、tが2〜3.5はセル2から画面外へ線形に移動。t >= 3.5f で FinishObstacle。GetOutsidePointの entering:false 側を進行方向の先（画面外）に配置し直す。

C8. 4方向対応 GetPathCellに Left と Down のケースを追加:

Left: lane=行、セル順は col 0→1→2（左から入って右へ通過）
Down: lane=列、セル順は row 0→1→2（上から入って下へ通過）
GetOutsidePointも同様に4方向対応。ResolveObstacleTextureで Left は _texObstacleRight(FlipH)、Down は _texObstacleBottom を使用。

D. 見た目改善
D1. Tweenイージング変更 TryMoveのTweenを .SetEase(Tween.EaseType.InOut).SetTrans(Tween.TransitionType.Cubic) に変更。

D2. アニメFPS低減 BuildPlayerFrames内: idle 8→5fps、run 10→6fps、hurt 10→6fps、death 8→5fps。move系は8fpsのまま。guard は 6fps。

D3. 背景3層化+スクロール _groundLayer（TextureRect）を動的生成し Backgroundに追加。_skyBgは y:0-960、_midGroundは y:300-1100 で2倍幅にして _Processで毎フレーム30px/s左スクロール（ループ）、_groundLayerは y:1050-1420でグリッド下の地面として表示。テクスチャは _texMidGround を流用（ground用テクスチャが無い場合は ColorRect で茶色 new Color(0.35,0.25,0.15,1) で代替）。

D4. 全UIテキスト日本語化

_stateLabel Idle時: "タップでスタート"
_stateLabel Playing時: "" (空)
_stateLabel Dead時: "DEAD - タップでリトライ"
_stateLabel Cleared時: "全Wave突破！"
_comboLabel: "コンボ: {_combo}"
_scoreLabel: "スコア: {_totalScore}"
_waveLabel: "Wave {n}/{total}"（これはそのまま）
Speakの各メッセージは既に日本語のため変更なし
BeginWave内の _stateLabel.Text = "Playing" → _stateLabel.Text = ""
D5. キャラスケール調整 _player.Scale を new Vector2(0.59f, 0.59f) → new Vector2(0.72f, 0.72f) に変更。

E. tscnの変更は不要
グリッドセルの位置はコード側の BuildGridCache で上書きしているため、GridSample.tscn の変更は不要。ただし BuildGridCache内で各 _cellBorders[col,row] の OffsetLeft/Top/Right/Bottom をコード側で明示的にセットするコードを追加する（現在はtscnの値を使っているが、CellSize/CellGap/GridOriginが変わるため齟齬が出る）。
